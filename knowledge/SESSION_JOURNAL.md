- **The security catch:** the specified `SECURITY DEFINER ingest_upsert_roles` "called via RPC after CRON_SECRET verification" would have been **grantable only to `anon`** (cron has no admin JWT; service-role banned) → **callable by anyone**, since the anon key ships in the browser bundle, and definer bypasses the `roles` admin-write RLS by design. The route-level secret check is irrelevant to an attacker POSTing at `/rest/v1/rpc/…`. **We had proven this exact mechanism 30 minutes earlier** — the session's own `rpc/get_applications` curl used nothing but the public anon key. Replaced with a bot auth user (option A) → `runIngest(client)` unchanged, one write path, zero new surface.
- **Self-correction mid-design:** I sold option A partly on "keep the bot out of `ADMIN_EMAILS` so a leak gives no admin access" — then read `config.ts:5-7` and found that mitigation **cosmetic** (`ADMIN_EMAILS` drives the UI; RLS is gated by `is_admin()`). Proposed `is_ingest_bot()` instead — ~15 lines more SQL than promised, but real least privilege. Told the user rather than quietly shipping the weaker version.
- **The real bug (worth more than the feature):** `classifyExpiry` had no notion of source + `fetchAllSources` is best-effort → one Adzuna 503 delists all ~44 Adzuna roles. The cron would have made it silent and nightly. Per-source scoping **alone** wouldn't fix it (a failed pull has an empty fresh set) — the fix is skipping errored sources, plus a circuit breaker for 200-with-zero-rows (failure mode: stale-and-visible, not silent loss).
- **Two traps caught by reading code, not assuming:** (a) `dedupe`'s richer-JD sort needed a `SOURCE_ORDER` fallback — stable sort would let input order decide ties and silently regress `pipeline.test.ts:11`; (b) `track()` no-ops server-side, so the specified `track('ingest_run')` would have recorded **nothing** behind a dashboard we'd trust → `trackServer()`. Also verified the stage7 policy names (`"roles admin insert"`/`"roles admin update"`) rather than guessing — a wrong name makes `drop policy if exists` a silent no-op and the cron fails at 3am with an RLS error.
- **`tsc` earned its keep:** widening `bySource` to `SourceStat` broke the admin render site; fixed there and used the moment to surface the new circuit-breaker warnings in that panel too.
- **Branched (`stage12-cron-ingestion`)** rather than pushing to `main` like every prior stage — after this morning's prod incident, keeping a DB-dependent change off auto-deploy until its migration exists. **NOT merged; needs the user's bot user + migration + env vars** (`docs/VERIFICATION_STAGE12.md` §0).
